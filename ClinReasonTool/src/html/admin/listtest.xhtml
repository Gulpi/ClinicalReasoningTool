Tests of List Configurtion by Properties Player/Viewer

tests of how lists can be included in xhtmls, these are now configured by properties

standard(de): #{adminContext.getMyListUrlByMode("view","standard","de")} should be jsonp_de.json (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk)
standard(en): #{adminContext.getMyListUrlByMode("view","standard","en")} should be jsonp_en.json (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk)
standard(sv): #{adminContext.getMyListUrlByMode("view","standard","sv")} should be - (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk) but lists.view.standard.sv=-
standard(uk): #{adminContext.getMyListUrlByMode("view","standard","uk")} should be - (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk) but lists.view.standard.ok=-
nursing(de): #{adminContext.getMyListUrlByMode("view","nursing","de")} should be jsonp_n_de.json
nursing(en): #{adminContext.getMyListUrlByMode("view","nursing","en")} should be jsonp_n_de.json when en is not configured (lists.languages.nursing=de) and default is de for nursing (lists.default_languages.nursing=de)
context(de): #{adminContext.getMyListUrlByMode("view","context","de")} should be jsonp_c_de.json
context(en): #{adminContext.getMyListUrlByMode("view","context","en")} should be jsonp_c_en.json
context(pl): #{adminContext.getMyListUrlByMode("view","context","pl")} should be jsonp_c_en.json when lists.languages.context=en,de and lists.default_languages.context=en

Usage in JSF templates

authoring exp_boxes.xhtml

{adminContext.getMyListUrl(<type>,adminContext.patillscript.locale)} or
{adminContext.getMyListUrlByMode(<mode>,"standard",adminContext.patillscript.locale)}

  • where <type> := standard | context | nursing at this moment (is configurable)
  • where <mode> := edit | view at this moment

viewing boxes.xhtml

{crtContext.getMyListUrl(<type>,crtContext..patillscript.local)}

  • where <type> := standard | context | nursing at this moment (is configurable)
