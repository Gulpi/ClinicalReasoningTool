tests of how lists can be included in xhtmls, these are now configured by properties

standard(de): "#{adminContext.getMyListUrlByMode("view","standard","de")}" and should be "jsonp_de.json" (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk)
standard(en): "#{adminContext.getMyListUrlByMode("view","standard","en")}" and should be "jsonp_en.json" (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk)
standard(sv): "#{adminContext.getMyListUrlByMode("view","standard","sv")}" and should be "" (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk) but lists.view.standard.sv=-
standard(uk): "#{adminContext.getMyListUrlByMode("view","standard","uk")}" and should be "" (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk) but lists.view.standard.uk=-
standard(pl): "#{adminContext.getMyListUrlByMode("view","standard","pl")}" and should be "jsonp_pl.json" (lists.languages.standard=en,de,pl,sv,es,pt,fr,uk)
nursing(de): "#{adminContext.getMyListUrlByMode("view","nursing","de")}" and should be "jsonp_n_de.json"
nursing(en): "#{adminContext.getMyListUrlByMode("view","nursing","en")}" and should be ""
context(de): "#{adminContext.getMyListUrlByMode("view","context","de")}" and should be "jsonp_c_de.json"
context(en): "#{adminContext.getMyListUrlByMode("view","context","en")}" and should be "jsonp_c_en.json"
context(pl): "#{adminContext.getMyListUrlByMode("view","context","pl")}" and should be ""

Usage in JSF templates

authoring exp_boxes.xhtml

{adminContext.getMyListUrl(<type>,adminContext.patillscript.locale)} or
{adminContext.getMyListUrlByMode(<mode>,"standard",adminContext.patillscript.locale)}

  • where <type> := standard | context | nursing at this moment (is configurable)
  • where <mode> := edit | view at this moment

viewing boxes.xhtml

{crtContext.getMyListUrl(<type>,crtContext..patillscript.local)}

  • where <type> := standard | context | nursing at this moment (is configurable)
